# Google Apps Script Productivity Suite — Project Overview

**Generated:** 2026-03-28 | **Scan Level:** Exhaustive

## Executive Summary

A collection of personal productivity automation scripts built on Google Apps Script (GAS). The suite automates repetitive tasks across Gmail, Google Drive, Google Calendar, and Google Docs. Scripts deploy directly into a user's Google Workspace via a browser-based deployment UI or manual copy-paste. Open-sourced under the MIT License.

## Architecture Type

**Pattern:** Script collection with shared utilities and browser-based deployment layer.

Each script follows the same internal architecture:
1. `code.gs` — GAS-native entry point (runs in Google's server environment)
2. `config.gs` — User-editable configuration (generated by deploy UI or manual)
3. `src/index.js` — Extracted pure logic (testable in Node.js via Jest)
4. `tests/` — Jest unit tests importing from `src/index.js`

The deployment layer (`deploy/index.html`) is a standalone static HTML page that uses Google Identity Services OAuth to create and configure GAS projects in the user's account via the Apps Script REST API.

## Repository Structure

- **Monolith** — single repository, single `package.json`, shared test infrastructure
- **3 automation scripts** in `src/` (gmail-to-drive, calendar-to-sheets, calendar-to-briefing-doc)
- **1 deployment utility** in `src/deploy/` (programmatic GAS project creation)
- **1 browser deployment UI** in `deploy/` (static HTML + Playwright E2E tests)
- **1 legacy GAS installer** in `gas-installer/` (web app-based deployment, older approach)
- **Shared test utilities** in `test-utils/` (GAS global mocks, Jest setup)
