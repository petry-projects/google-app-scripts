🚀 Deploy Google Apps Scripts

Deploy a script from this repository directly to your Google account using your browser.

1Enter your OAuth Client ID

A Web application OAuth 2.0 client ID from a Google Cloud project is required. The client ID is used only to sign you in and request permission to create Apps Script projects — no credentials are sent to any server.

How do I get a Client ID?
  1. Open the Google Cloud Console and create or select a project.
  2. Go to APIs & Services → Library and enable the Google Apps Script API.
  3. Go to APIs & Services → OAuth consent screen. Set it to External (so anyone can use it). Under Scopes, add: https://www.googleapis.com/auth/script.projects
  4. Go to APIs & Services → Credentials and click Create Credentials → OAuth client ID.
  5. Choose Web application as the application type.
  6. Under Authorised JavaScript origins, add the origin you are running this page from (e.g. https://<your-username>.github.io or http://localhost).
  7. Click Create and copy the Client ID shown.
  8. Note your Project Number from the GCP dashboard (visible on the home page or under IAM & Admin → Settings). You will enter this into Settings → "Google Cloud Platform (GCP) Project" in your Installer GAS project.

2Sign in with Google

Sign in to the Google account where you want the script project created. You will be asked to grant permission to create and edit Apps Script projects.
